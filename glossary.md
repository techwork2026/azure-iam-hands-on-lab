# IAM Glossary

## Identity
A digital representation of a user in a system.

Example:
John Employee, Contractor, or Intern accounts in Microsoft Entra ID.

---

## Authentication
The process of verifying a user's identity.

Examples:
- Username and password  
- Multi-Factor Authentication (MFA)

---

## Authorization
Determining what a user is allowed to access after authentication.

Examples:
- Access to applications  
- Permissions within systems  

---

## MFA (Multi-Factor Authentication)
A security method that requires more than one form of verification.

Examples:
- Password + phone notification  
- Password + authentication app  

---

## RBAC (Role-Based Access Control)
A method of assigning access based on roles instead of individuals.

Examples:
- User Administrator → manage users  
- Global Administrator → full access  

---

## Least Privilege
A security principle where users are given only the access needed to perform their job.

This reduces risk and prevents unnecessary access.

---

## Sign-In Logs
Records of user authentication activity within the system.

Used to:
- Investigate login issues  
- Detect failed sign-ins  
- Monitor security activity  

---

## Account Enabled / Disabled
Controls whether a user can sign in.

- Enabled → user can access system  
- Disabled → user is blocked from signing in  

---

## Group Membership
Users can be added to groups to organize access.

Important:
Group membership alone does not grant permissions unless roles are assigned.

---

## Key Takeaway

IAM involves:
- Managing identities  
- Controlling access  
- Troubleshooting issues  
- Investigating security activity  

These terms represent core concepts used in real IAM roles.
